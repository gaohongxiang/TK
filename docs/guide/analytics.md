# 数据分析

数据分析用于本地导入 TikTok Shop 商品流量详情 Excel，快速看渠道表现、商品排行、流量漏斗和商品动作优先级。

这个模块会复用全站 Firebase 连接。Excel 原始文件只在当前浏览器里读取和解析，不上传到 Cloudflare；解析后的分析快照会写入你的 Firestore，刷新页面后可以恢复最近一次分析。

## 支持的文件

- TikTok Shop 导出的商品流量详情 Excel。
- 文件后缀支持 `.xlsx` 和 `.xls`。
- 表格中需要包含 `ID`、`商品`、`GMV`、`成交件数`、`订单数` 等商品流量字段。

如果页面提示没有找到表头，通常是因为导出的表不是商品流量详情，或表格结构被手工改过。

## 怎么使用

1. 打开主站的 `数据分析`。
2. 点击 `选择 Excel 文件`。
3. 选择 TikTok Shop 导出的商品流量详情表。
4. 页面会在本地解析文件，并生成 KPI、渠道 GMV、流量漏斗、周期对比、Top 商品、动作优先级和商品明细。

连接 Firestore 后，导入完成会保存分析快照；刷新页面会自动恢复最近一次分析。未连接数据库时仍可临时生成当前页面看板，但不会保存。

## 指标口径

- `GMV`：读取表格里的商品 GMV。
- `成交件数`：读取商品成交件数。
- `订单数`：读取商品订单数。
- `曝光`：商城、视频、商品卡、直播等渠道曝光加总。
- `浏览`：各渠道页面浏览次数加总。
- `去重客户数`：各渠道去重客户数加总。
- `CTR`：浏览次数 ÷ 曝光次数。
- `转化率`：成交件数 ÷ 去重客户数。

渠道维度当前覆盖：

- 商城
- 视频
- 商品卡
- 直播

## 动作优先级

系统会按 GMV、订单数、曝光、浏览、点击率、转化率和多周期趋势给商品生成动作优先级。诊断标签仍保留在商品明细里，动作优先级用于告诉你先处理哪几个商品。

- `优先放大`：GMV 或订单数较高，优先补库存、加素材、配优惠活动。
- `补流量测试`：已有成交但曝光不足，优先补短视频和商品卡曝光。
- `换素材`：曝光不低但点击弱，优先换主图、标题关键词和视频前三秒。
- `改详情/价格`：有浏览无成交，检查价格、评价、详情页卖点和优惠券。
- `暂停观察`：低曝光且无成交，短期补素材测试，仍无起量则下架或换品。
- `继续观察`：暂不需要特殊动作，继续监控。

这些动作建议是运营辅助判断，不是平台官方结论。真正决策时还要结合库存、利润、类目周期、达人素材和活动节奏。

## 和商品管理的关系

数据分析不会自动写入商品管理。分析快照会保存到 Firestore 的数据分析集合，用于恢复看板；商品管理里的商品资料仍需手动维护。

如果你想把分析结论沉淀成商品动作，需要手动回到商品管理或订单管理里调整商品信息、SKU 参数、库存计划或运营备注。

## 数据边界

- 不上传 Excel。
- 不保存 Excel 原始文件。
- 解析后的分析快照写入你自己的 Firestore。
- 不把分析结果发送到第三方接口。
- 未连接 Firebase 时只临时显示当前页面看板。

这个模块适合临时复盘一段时间的商品流量表现，不适合作为长期数据仓库。
