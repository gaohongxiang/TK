# 数据库

商品管理、订单管理、数据采集和数据分析都依赖你自己的 `Firebase Firestore` 项目。第一次使用时，先把数据库创建好、规则发布好，再回到网站连接。

当前正式数据源是 Firebase-only。页面、文档和构建产物不再把 Supabase 当作当前推荐方案；仓库里如果还保留历史 Supabase 文件，只能视为实验记录或未启用代码。

## 为什么选择 Firebase

- **数据在你自己手里：** 商品资料、订单资料、采集记录和数据分析快照都保存到你自己的 `Firebase Firestore` 项目里。本站不保存你的数据。
- **接入简单：** 这是一个纯前端工具，不需要你自己先搭后端服务，创建好 Firebase 项目后就可以直接连接使用。
- **字段扩展更轻：** Firestore 是文档型数据库。后续如果只是给订单或商品增加字段，大多数情况下不需要像传统 SQL 表那样先改表结构再继续用。
- **离线和同步现成：** 网站使用 `Firestore 自带的离线缓存`。离线时也能继续操作，恢复联网后会自动同步。
- **换设备恢复简单：** 只要新设备连接的是同一个 Firebase 项目，并填入同一套 `firebaseConfig`，就可以恢复数据。
- **一处连接，全站复用：** 商品管理、订单管理、数据采集和数据分析共用同一个 Firestore 项目。连一次，相关模块都会直接复用。
- **更适合自己部署：** 如果你把项目部署到 Cloudflare Pages，也可以继续连你自己的 Firebase 项目，不需要把数据托管给别人。

## 哪些数据会写入哪里

| 数据 | 保存位置 | 说明 |
| --- | --- | --- |
| 商品、SKU、账号 | 你的 Firebase Firestore | 商品管理保存和读取 |
| 订单、订单明细、物流信息 | 你的 Firebase Firestore | 订单管理保存和读取 |
| 采集商品、目标账号、采集状态、拒绝品去重 | 你的 Firebase Firestore | 数据采集保存和读取；同一账号下按商品去重，不同账号互不覆盖 |
| 商品流量 Excel 原始文件 | 浏览器内存 | 数据分析导入后本地解析，不上传 |
| 数据分析快照和商品明细 | 你的 Firebase Firestore | 数据分析保存和读取，刷新后恢复最近一次分析 |
| 汇率、运费倍率、连接配置 | 当前浏览器 localStorage | 本地工具参数，不是平台方数据库 |
| 静态页面、JS、CSS | Cloudflare Pages | 只托管前端资源 |

Cloudflare Pages 不保存你的商品、订单、采集记录、Excel 原始文件或分析快照。

## 创建 Firebase 项目

1. 打开 [Firebase Console](https://console.firebase.google.com/)。
2. 新建一个项目。
3. 添加应用时选择 `网页`。
4. 不用勾选 `Firebase Hosting`。
5. 复制系统生成的整段 `firebaseConfig`。

## 创建 Firestore 数据库

1. 在 Firebase 项目里进入 `Firestore Database`。
2. 创建数据库时建议选择：
   - `区域级`
   - `生产模式`
3. 创建完成后，进入 `Rules` 页面。

## 发布 Firestore 规则

本网站提供了最新的 Firestore 规则。你需要把规则复制到 [Firebase Console](https://console.firebase.google.com/) 里并发布。

- 第一次使用时，需要先发布一次规则。
- 以后如果页面提示“需要更新 Firestore 规则”，也要重新复制并发布一次。

## 在网站里连接数据库

1. 回到网站，进入 `商品管理` 或 `订单管理`。
2. 点击 `连接 Firebase`。
3. 粘贴整段 `firebaseConfig`。

::: tip
连接成功后，商品管理、订单管理、数据采集和数据分析会直接共用这一次连接，不需要分别再配一次数据库。
:::

## 网站如何判断规则是否最新

网站不能直接读取你 Firebase 后台里的规则文本，所以会用当前模块的真实读取结果判断规则是否可用：

- 用户进入商品管理，就读取商品管理需要的数据。
- 用户进入订单管理，就读取订单管理需要的数据。
- 用户进入数据采集，就读取数据采集需要的数据。
- 用户进入数据分析，就读取最近一次分析快照。
- 真实读取成功就静默，不弹提示。
- 真实读取失败并且是权限问题时，才在当前模块列表区显示权限不足提示。
- 不会为了检查权限额外写入商品、订单或采集数据。
- 保存、删除、更新这类写入权限，只在用户真实执行对应操作失败时提示。
- 这个判断不会上传到本站，也不会改变业务数据结构。

如果规则太旧，页面顶部只保留 `已连接 · 项目名` 这类连接状态。列表区会显示“数据库权限不足”和复制最新规则、打开 Firebase Console 的按钮；不会弹全局长说明，不会列技术清单，也不会把内部集合名展示给用户。

各模块列表区统一按四种状态显示：

- 未连接数据库：显示连接数据库按钮。
- 已连接但当前模块权限不足：显示权限不足和复制规则按钮。
- 已连接且权限正常但没有数据：显示“还没有数据”一类空状态。
- 已连接且有数据：显示当前模块表格。

如果以后新增模块，这个模块只需要在自己的真实读取和写入失败处接入同一套人话提示。用户进入这个模块时，只会触发这个模块自己的读取；其他模块不会被顺手检查，也不会因为无关模块缺权限而弹提示。

最新规则里保留 `_tk_probe` 路径，后续如果需要做专门的规则版本检查可以使用；当前页面不会为了进入模块而写这个路径。真实业务数据仍然放在商品、订单、数据采集和数据分析各自的数据集合里。

## 账号如何保存

商品管理、订单管理和数据采集共用同一套账号标签。账号标签保存在统一账号表里，三个模块读取同一份列表。

- 在任意模块新增账号，其他模块会同步看到。
- 账号标签顺序以统一账号表为准，先添加的在左边，后添加的在右边；拖动排序后三个模块共用这个顺序。
- 商品、订单和采集记录里的账号字段只用于计数和筛选，不会反向生成新的账号标签。
- 如果某条数据没有账号，或对应账号标签后来被删除，这条数据不会丢失，只会在“全部”里显示。
- 具体账号标签上可以打开账号菜单，编辑账号名或删除账号名。
- 编辑账号名会同步更新商品、订单和采集记录的账号归属。
- 删除账号名只删除账号标签，不删除商品、订单或采集记录；删除后这些数据只在“全部”里显示。

## 如何退出当前数据库连接

- 连接成功后，`商品管理`、`订单管理` 和 `数据采集` 顶部状态条里都会显示 `退出数据库` 按钮。
- 点击后会断开当前 Firebase 项目连接。
- 断开后，相关模块会一起回到未连接状态；下次需要重新连接时，再粘贴 `firebaseConfig` 即可。

## 自己部署

Cloudflare Pages 主站和文档站的配置、发布前检查、CI 门禁和部署后验收集中放在 [部署与发布检查](/guide/deploy)。

## 什么时候需要更新规则

通常不是每次加字段都要重新发布规则。一般在以下情况需要更新：

- 新增了新的集合。
- 权限模型发生变化。
- 页面弹出了“需要更新 Firestore 规则”的统一提示。

如果需要更新规则，直接打开 [Firebase Console](https://console.firebase.google.com/)，复制最新规则，在 `Firestore Rules` 页面重新发布即可。
