# 收支管理

收支管理用于把订单里的预估利润和真实现金流分开看。订单利润是复盘口径，收支记录是真实回款、运营成本和押金占用。

收支数据保存到你自己的 Firebase Firestore。本站不保存你的回款、成本或押金记录。

## 两套口径

- **预估口径：** 预估收入直接取订单管理当前筛选范围内的预估利润，再扣手工录入的运营成本。
- **真实口径：** 实际净额 = TK 提现回款 - 订单成本 - 运营成本。订单成本由采购价和当前全局贴单费组成，不再扣订单预估运费。

## 记录什么

- `回款`：例如 TK 提现、其他回款、押金退回。
- `成本`：例如开店成本、IP 成本、投流成本、软件订阅、样品费、手续费、押金、押金扣除、其他。
- `押金`：表示资金占用，不算运营成本。
- `押金扣除`：表示罚扣或损失，会算入运营成本。

## 账号和公共账

- 顶部账号标签会先筛选收支记录和订单，再计算汇总。
- `全部` 会看所有账号和公共账。
- `公共账` 只看没有账号归属的收支记录，不包含订单。
- 某个具体账号只看该账号的收支记录和订单。

## 搜索和月份

- 月份默认看全部，可以切到指定月份。
- 搜索框支持日期、类别、备注、金额和账号关键词。
- 当前筛选结果会同时影响表格、预估口径和真实口径。

## 保存和同步

收支记录保存到 `finance_records` 集合。保存、删除和账号改名会更新收支管理同步状态；如果 Firestore 规则太旧，页面会提示复制最新规则。
