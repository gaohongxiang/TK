# 订单管理

订单管理用于记录采购、物流、入仓进度，并汇总销售额、支出和预估利润。

订单数据保存到你自己的 Firebase Firestore。本站不保存你的订单、物流单号或利润数据。

## 新增订单时，推荐怎么录

1. **先填基本信息：**
   - `账号`
   - `下单时间`
   - `订单号`
   - `采购日期`
2. **再填订单明细项：**
   - 逐条添加商品 / SKU / 数量
   - 每条订单明细对应一个商品的一个 SKU
3. **再看订单级汇总：**
   - `总件数`：按所有明细数量自动求和
   - `总重量`：按所有明细 `单件重量 × 数量` 自动汇总
   - `总尺寸`：按实际打包情况自己调整
   - `预估总海外运费`：按当前的 `总重量 + 总尺寸` 自动计算
4. **再填订单级金额：**
   - `总售价（日元）`
   - `达人佣金率（%）`
   - `达人佣金（元）`
   - `总采购额（元）`
   - `是否退款`
5. **最后补订单级状态：**
   - `订单状态`
   - `订单预警`

## 订单明细怎么理解

一张订单里可以有多条明细，每条明细代表：

- 一个商品的一个 SKU
- 这个 SKU 的数量
- 这条明细自己的快递公司和快递单号

订单可以不关联商品直接手工录入，但推荐先在商品管理里维护商品和 SKU。关联后，订单会自动带出商品名称、单件重量、单件尺寸和价格快照，减少重复输入。

## 多条明细的快递信息怎么录

- 每条明细都可以填自己的 `快递公司 / 快递单号`
- 如果多个商品共用同一个单号，可以先把第一条填好，再点 `快递单号` 旁边的复制图标
- 复制后，去另一条明细里正常粘贴单号即可；快递公司会继续按单号自动识别

## 订单里的金额口径

- `总售价`：日元
- `达人佣金率`：百分比
- `达人佣金`：人民币
- `总采购额`：人民币
- `预估总海外运费`：人民币
- `预估利润`：人民币

预估利润公式：

`预估利润 = 实收金额（日元）÷ 汇率 -（总采购额 + 预估总海外运费 + 达人佣金）`

## 保存和同步

订单保存时会先进入 Firestore SDK 的本地队列，页面立即更新，不等待云端提交完成。这样可以保持录单速度；如果暂时离线，恢复联网后 Firebase SDK 会继续同步。

当前订单同步核心保持稳定，尤其不要回退快递单号输入框焦点和自动识别逻辑。改订单保存、快递公司或快递单号相关功能时，要重点手动检查新增订单、编辑订单、复制单号和自动识别。

## 达人订单怎么识别

- 只要这张订单填了 `达人佣金率`，或已经算出了 `达人佣金`，系统就会把它当作达人带货订单。
- 在订单列表里，订单号后面会显示一个浅色 `达人` 标记，方便你一眼区分普通订单和达人单。
- 搜索框里直接输入 `达人` 或 `达人单`，也能筛出这类订单。

## 退款订单怎么处理

订单标记为退款后，页面会把销售额、佣金和利润相关口径按退款逻辑处理，列表里也会出现退款标识。退款订单仍会保留采购、物流和历史记录，方便后续复盘。

## 账号筛选和搜索

- 顶部账号标签可以按店铺账号筛订单。
- 搜索框支持订单号、产品、状态、快递公司、快递单号等关键词。
- 当前筛选结果会影响列表和统计卡片，方便按账号或关键词复盘。
