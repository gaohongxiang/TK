# 概览

本文档分成两部分：前半部分是工具使用，后半部分是 TK 运营里的选品、内容发布、营销活动和话术。

TK 电商工具箱当前定位是“静态工具站 + 用户自有数据源”。网站提供利润计算、商品管理、订单管理和本地数据分析能力，但不托管你的商品、订单或 Excel 文件。

- 项目已开源：[https://github.com/gaohongxiang/TK](https://github.com/gaohongxiang/TK)
- 你可以直接基于开源代码自行部署到 `Cloudflare Pages`
- 利润计算器不依赖数据库，可以直接使用
- 商品管理和订单管理依赖你自己的 Firebase Firestore 项目，本网站不保存你的数据
- 数据分析只在浏览器本地解析 Excel，不上传文件，也不写入数据库

## 当前模块

- `利润计算器`：用于定价、利润复盘、海外运费估算。
- `商品管理`：维护账号、商品、SKU、重量尺寸和预估运费。
- `订单管理`：记录订单明细、采购、物流、入仓状态、达人佣金和预估利润。
- `数据分析`：导入 TikTok Shop 商品流量 Excel，在浏览器内生成渠道表现、商品排行和诊断标签。详见[数据分析](/guide/analytics)。
- `文档站`：说明工具使用、Firebase 连接、部署方式和 TK 运营资料。

## 数据边界

- `商品` 和 `订单` 写入你自己配置的 Firebase Firestore。
- `Excel` 只在当前浏览器页面内解析，刷新页面后不会保留。
- `localStorage` 只保存汇率、运费倍率、连接配置等本地工具参数。
- `Cloudflare Pages` 只托管静态文件，不保存用户业务数据。

## 推荐上手顺序

1. 先连接 Firebase。
2. 在商品管理或订单管理里建立账号都可以。
3. 在商品管理里录入商品和 SKU。
4. 录订单时优先关联商品和 SKU。
5. 需要复盘商品流量时，再导入 TikTok Shop 商品流量 Excel。
6. 利润计算器想什么时候用就什么时候用。

## 自己部署时怎么理解

主站和文档站建议拆成两个 Cloudflare Pages 项目：

- 工具主站：根目录构建，命令 `npm run build`，输出 `dist`。
- 文档站：`docs` 目录构建，命令 `npm run build`，输出 `.vitepress/dist`。

两个站都只是静态发布。即使你使用公开 Pages 域名，用户业务数据仍然只进入用户自己的 Firebase 项目或浏览器本地内存。
