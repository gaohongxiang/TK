# 商品管理

商品管理用于维护商品主档、SKU 等，目的是让后续录订单时可以直接复用，不再重复填写。

商品数据保存到你自己的 Firebase Firestore。网站只提供前端页面和交互逻辑，不保存你的商品资料。

## 商品主档里维护什么

- 所属账号
- TK ID
- 商品名称
- 1688 链接
- 图片链接
- 货物类型
- SKU 列表

## SKU 里维护什么

- SKU 名称
- SKU ID
- 是否继承商品共用物流参数
- 单件重量
- 单件尺寸
- 预估海外运费
- 计费重和运费提示

## 新增商品时，推荐怎么录

1. 先填商品主档。
2. 再新增 SKU。
3. 最后做参数调整。
4. 保存前检查每个 SKU 是否都有重量和尺寸。

## 导出 CSV 怎么用

- 商品管理右上角提供 `导出 CSV` 按钮。
- 导出时会直接按你当前页面的筛选结果导出：
  - 当前账号标签
  - 当前搜索关键词
  - 当前排序结果
- 导出的口径是 **一条 SKU 一行**，方便后续继续整理、复核或交给别人使用。

## SKU 怎么新增

- **新增单个 SKU：** 适合只有一个规格，或者只想手动补一个新 SKU。
- **新增多个 SKU：** 适合颜色、尺寸、款式这类组合规格场景。

例如输入：

- 规格维度 1：`白、黑、蓝`
- 规格维度 2：`S、M、L`
- 规格维度 3：`普通款、升级款`

系统会自动生成：

- `白 / S`
- `白 / M`
- `白 / S / 普通款`
- `黑 / L`
- `蓝 / S`

## 参数调整怎么用

- **所有 SKU 的重量和尺寸都一样：** 先生成全部 SKU，`匹配关键词` 留空，直接填写重量和尺寸，系统会同步到全部 SKU。
- **按分组不同：** 例如按 `S / M / L` 分组，先生成全部 SKU，`匹配关键词填 S`，同时填写重量和尺寸，系统会同步到所有匹配到 S 的 SKU，没命中的 SKU 原来的值保持不变。
- **各 SKU 的重量尺寸输入框也可单独修改：** 比如其中某一两个 SKU 需要单独修正。

尺寸建议按 `长×宽×高` 填写，例如 `20×15×10`。输入 `20*15*10` 也会被识别并统一成乘号格式。尺寸不足三段时，系统只能按实重粗估，无法完整校验体积重。

## 离线和保存

商品保存时会先进入 Firestore SDK 的本地队列，页面会先更新，不需要等待云端提交完成。断网时也可以继续操作；恢复联网后由 Firebase SDK 自动同步。

如果 Firestore 规则没有放行 `products` 集合，页面会提示更新规则。按提示复制最新规则到 Firebase Console 发布即可。

## 商品和订单管理怎么联动

- 新增订单时可以关联商品，以及继续选到具体的 SKU。
- `关联商品` 下拉支持按 `商品 ID / 商品名称` 搜索。
- `关联 SKU` 下拉支持按 `SKU ID / SKU 名称` 搜索。
- 关联后，系统会自动带出商品名称、单件重量和单件尺寸。
- 订单保存时会写入自己的快照，所以后面即使你改了商品或 SKU，历史订单也不会被反向改乱。

## 导入数据分析和商品管理的关系

数据分析模块读取的是 TikTok Shop 导出的商品流量 Excel，解析结果只留在当前浏览器页面内。它不会自动写入商品管理，也不会把 Excel 上传到 Firestore。

如果你想把数据分析里的发现沉淀到商品管理，需要手动回到商品管理修改商品信息。
