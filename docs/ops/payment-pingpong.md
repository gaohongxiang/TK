# PingPong 收款绑定

这页整理 TikTok Shop 店铺绑定 PingPong 收款账号时的操作要点。核心原则是：**TikTok 店铺主体、PingPong 持有人和后续提现账户应保持真实、可核验、可解释**。

原始资料里提到“错名绑定”。实际运营中不建议把它理解成规避实名一致性。更稳的做法是：如果一个 PingPong 账号需要服务多个店铺主体，就在 PingPong 的持有人管理里维护对应主体资料，绑定店铺时选择与该 TikTok 店铺主体匹配的持有人。

## 适用场景

- 一个团队管理多个 TikTok Shop 店铺。
- 不同店铺对应不同个人或企业主体。
- 希望在一个 PingPong 后台里集中管理多个店铺的收款账号。
- 需要区分不同店铺主体，避免绑定时选择错误持有人。

## 准备资料

绑定前先确认这些信息：

- TikTok Shop 店铺主体：个人或企业，主体名称必须能和平台资料对应。
- PingPong 实名主体：想提现到谁的账户，就用谁的真实资料完成认证。
- 店铺对应持有人资料：个人身份证明或企业证照、受益人/法人等平台要求资料。
- 后续提现银行卡：应与 PingPong 认证主体和资金归属关系匹配。

## PingPong 注册和实名

1. 打开 [PingPong 注册入口](https://us.pingpongx.com/entrance/signup?cb=true&inviteCode=ENnym2246)。
2. 按实际经营主体完成实名认证，个人认证和企业认证都可以。
3. 如果页面要求填写客户经理，填写：`付有康`。
4. 认证完成后再处理 TikTok 店铺绑定，不要先用临时或不确定资料绑定。

::: tip 操作原则
谁收款、谁提现，就用谁或对应企业的真实资料认证。不要为了通过页面校验随意填写不属于自己的主体资料。
:::

## 多店铺持有人管理

当一个 PingPong 账号里需要绑定多个 TikTok 店铺主体时，可以按下面方式管理：

1. 进入 PingPong 右上角的账号中心。
2. 打开“持有人管理”。
3. 点击新增持有人。
4. 按目标 TikTok 店铺主体上传对应资料：
   - 绑定 B 公司店铺，就维护 B 公司资料。
   - 绑定 C 公司店铺，就维护 C 公司资料。
   - 个人店铺就维护对应个人资料。
5. 如果页面允许跳过持有人银行卡信息，可先跳过，后续按提现需求补齐。
6. 提交后等待审核通过。

这里的关键不是“名字不同也强行绑定”，而是让 PingPong 里存在一个与目标店铺主体对应的持有人。后续绑定时选择正确持有人，资金归属和平台资料才容易解释。

## 绑定 TikTok Shop 店铺

1. 在 PingPong 右上角进入“申请收款账号”。
2. 平台选择 `TikTok`。
3. 如果是欧洲店铺，选择对应的 TikTok 欧洲入口。
4. 点击前往 TikTok 后台绑定。
5. 选择与 TikTok 店铺主体一致的 PingPong 持有人。
6. 按跳转流程完成授权和绑定。
7. 绑定后回到 TikTok Shop 后台确认收款账号状态。

## 常见检查

绑定前检查：

- TikTok 店铺主体和 PingPong 持有人是否对应。
- 店铺地区和 PingPong 申请入口是否对应，例如欧洲店铺不要选错地区。
- 提现银行卡是否属于实际收款主体。
- 多店铺时是否选错持有人。

绑定后检查：

- TikTok 后台收款账号是否显示正常。
- PingPong 后台是否能看到对应店铺。
- 后续提现路径是否清楚。
- 店铺主体、持有人、收款账户之间是否有可解释关系。

## 风险提醒

- 不要使用不属于自己的个人或企业资料。
- 不要为了降低账号关联风险而虚构主体、地址、联系人或收款信息。
- 如果 TikTok 后台提示持有人不一致，应优先检查店铺主体和 PingPong 持有人是否选错。
- 多店铺集中管理时，建议建立一张内部表，记录“店铺名称 / 店铺主体 / PingPong 持有人 / 收款账号 / 提现账户”。
- 收款资料属于高风险资料，变更前最好截图留档，方便后续排查。

## 内部记录模板

| 字段 | 填写内容 |
| --- | --- |
| TikTok 店铺名称 | 店铺后台显示名称 |
| 店铺地区 | 日本 / 欧洲 / 其他 |
| 店铺主体 | 个人姓名或企业名称 |
| PingPong 持有人 | 绑定时选择的持有人 |
| PingPong 收款账号 | 后台生成的收款账号标识 |
| 提现账户 | 银行卡或企业账户 |
| 绑定日期 | 完成绑定的日期 |
| 备注 | 审核状态、客户经理、异常记录 |
