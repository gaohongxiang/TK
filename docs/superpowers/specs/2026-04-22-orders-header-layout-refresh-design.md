# 订单页顶部布局重组设计

日期：2026-04-22

## 目标

在不改变现有页面风格的前提下，重组订单页表格上方的布局层级，让页面更接近正规的数据后台界面：

- 主次动作分离
- 统计、筛选、分页职责分层
- 账号很多时布局仍稳定
- 桌面端和移动端都保持可用

本次只调整订单页顶部布局，不改变订单表格字段、统计口径和现有业务规则。

## 设计方向

整体风格保持当前页面的配色、圆角、按钮和卡片语言，不引入新的视觉主题。优化重点放在信息架构和交互路径：

- 全局状态放最上层
- 统计放在状态下方，作为页面概览
- 账号切换和新增订单绑定在同一层
- 搜索和分页归为表格控制带，并吸顶

## 桌面端布局

订单页表格上方重组为四层：

### 第一行：状态栏

左侧显示：

- 当前存储目标
- 同步状态
- 数据存储说明入口

右侧显示全局辅助动作：

- 刷新
- 导出
- 退出

这里不再放“新增订单”，避免主动作和辅助动作混在一起。

### 第二行：统计卡片

保留两张卡片，并排显示：

- 当前筛选采购总额
- 全部订单采购总额

两张卡片先保持等宽并排，方便后续扩展为更多统计卡片，不提前做强主次布局。

### 第三行：账号标签行

这一行采用三段结构：

1. 左侧固定 `全部`
2. 中间其他账号横向滚动
3. 右侧固定两个按钮：
   - 添加账号
   - 新增订单

约束如下：

- `全部` 不参与横向滚动，始终固定在左侧
- 其他账号不再整体换行，避免把右侧主按钮挤下去
- `新增订单` 是这一行的主按钮，视觉上独立于账号标签
- `添加账号` 固定在右侧，位于 `新增订单` 左边，不跟随账号区滚动

### 第四行：吸顶控制带

将以下控件归入同一条表格控制带：

- 搜索框
- 每页条数
- 上一页 / 页码 / 下一页

布局方式：

- 搜索框靠左
- 每页条数和分页控件靠右
- 整条控制带在滚动表格时吸顶

吸顶样式采用整条面板色固定吸顶，不做独立悬浮胶囊。

原因：

- 更像正式后台列表控制区
- 滚动时仍能快速搜索和翻页
- 不需要把整个头部固定，避免页面压迫感过强

## 移动端布局

移动端延续桌面端层级，但改为更松的两层布局，避免控件硬挤一行：

### 顶部层级

- 第一行：状态栏保留，但动作区允许换行
- 第二行：统计卡片上下堆叠
- 第三行账号区改成两层：
  - 第一层：`全部` + 其他账号横向滚动
  - 第二层：`添加账号 + 新增订单`

### 吸顶控制带

移动端的吸顶控制带拆成上下结构：

- 上层：搜索框
- 下层：每页条数 + 分页控件

这样可以减少拥挤，同时保留吸顶带来的查单效率。

## 交互规则

### 新增订单

- `新增订单` 按钮固定在账号标签行右侧
- 用户先看到当前账号范围，再点击新增
- 这让“当前筛选上下文”和“新增动作”保持邻近关系

### 账号很多时

- 其他账号区域只横向滚动，不自动扩成多行
- `全部`、`添加账号`、`新增订单` 始终固定可见
- 暂不增加“更多”或“搜索账号”入口，等账号规模真的增长后再补

### 搜索与分页

- 搜索、每页、分页全部归入同一条控制带
- 控制带吸顶后，表格滚动时仍保留主要查找动作

## 不在本次范围内

- 不修改采购统计口径
- 不增加新的统计项
- 不增加账号搜索或“更多账号”入口
- 不修改订单表格字段和列顺序
- 不修改存储逻辑、同步逻辑和迁移逻辑

## 实施提示

实现时应尽量沿用现有组件和 CSS 命名，不推翻当前结构。重点改动应集中在：

- 顶部栏容器拆分
- 账号标签区滚动容器
- 右侧固定动作区
- 控制带吸顶样式
- 移动端断点布局

## 验收标准

- 桌面端顶部四层结构清晰可见
- `新增订单` 不再位于第一行工具栏
- 账号很多时不会把 `新增订单` 挤掉
- `全部` 始终固定在账号区左侧
- 搜索和分页整条控制带吸顶
- 移动端不出现明显拥挤和换行错乱
- 页面整体视觉语言与当前版本保持一致
