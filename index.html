定价旧 / 定价新 / 利润复盘有什么区别？

定价旧
旧版定价逻辑，独立使用旧口径的采购价、手续费、汇率和运费参数来反推原价。它不和定价新、利润复盘共用逻辑，后续也可能被移除。
定价新
新版定价逻辑，按采购价、海外运费和新版运费口径来计算总费用，再结合折扣、平台手续费和汇率反推原价。
利润复盘
适合订单已经成交、商品售价已经确定时使用。按新版定价口径，基于采购价、海外运费和实际售价，直接复盘人民币到手、利润和利润率。

选择订单存储方式

为了避免数据丢失并支持跨设备恢复，必须选择一个云端存储。Gist 模式会同步到你自己的 GitHub 私有 Gist；Firestore 模式会同步到你自己的 Firebase 项目，并使用 Firestore 自带的离线缓存。

Gist 模式会在本地缓存之外，把订单同步到你自己 GitHub 账号下的私有 Gist，别人看不到，也方便跨设备同步。首次使用需要生成一个仅带 gist 权限的 Token。

如果你以后要在新浏览器或新设备连回原来的旧数据，请把 Token 和 Gist ID 一起保存好；私有 Gist 需要两者同时具备，才能重新连接到同一份数据。

订单会按账号拆分同步到 Gist。通常各自只操作自己的账号时，互相不会影响；同账号下如果改的是不同订单，会自动合并；如果两边同时改了同一条订单，同步时会让你选保留本地还是云端。

  1. 打开 GitHub Token 生成页（已预填权限）
  2. 确认只勾选 gist，有效期按需选（建议 90 天及以上）
  3. 点底部 Generate token，复制 ghp_ 开头的 Token 粘贴到下方
  4. 首次会自动创建私有 Gist；如果你已经有数据 Gist，把它的 ID 填到下方，并把 Token 和 Gist ID 一起保存好

Firestore 模式下，订单数据保存在你自己的 Firebase 项目里。你只需要准备好自己项目的 firebaseConfig。

连接后会直接使用 Firestore 自带的离线缓存，换设备时只要填同一个项目的 firebaseConfig 即可恢复数据。

Firestore 三步接入
  1. 新建项目，添加网页应用 打开 Firebase Console 新建项目，添加应用时选 网页，不用勾 Hosting。
  2. 创建 Firestore 数据库并发布规则 进入刚刚创建的项目，在左侧边栏中选择 数据库与存储 ,选择 Firestore 按钮创建数据库；位置建议选 区域级 - HongKong，模式选 生产模式。建好后到 Rules 页面，粘贴下方按钮复制出来的规则并发布。
  3. 粘贴 firebaseConfig 回来连接 把控制台生成的整段 firebaseConfig 粘到下方；有旧 Gist 数据就点迁移，没有就直接连接。
团队成员可以共用同一个 Firebase 项目，只要使用同一套 firebaseConfig 即可；当前方案不做成员级权限隔离，更适合彼此信任的小团队或自用场景。
已有 GitHub Gist 数据？迁移源始终是 Gist 云端，不会读取当前 Firestore 里的数据；如果你后来只在 Firestore 里新增过订单，迁到新库时不会自动带过去。

数据存储说明

本地优先
Gist 模式下，订单会先写入当前浏览器本地数据库（IndexedDB），随后再同步到你自己的 GitHub Gist；Firestore 模式下，则直接使用 Firestore 自带的离线缓存与云端同步。
可选远端
Gist 模式会同步到你自己的 GitHub 私有 Gist；Firestore 模式会同步到你自己的 Firebase Firestore。工具本身不会把订单存到我的数据库里。
恢复方式
如果你使用 Gist，请保存好 Token 和 Gist ID；如果你使用 Firestore，请保存好自己的 firebaseConfig。换浏览器或换设备后，用同一套远端配置即可恢复。
团队共用
同一个 Firebase 项目可以给团队成员共用，但当前方案没有成员级权限隔离；拿到这套 firebaseConfig 的成员都能读写全部订单。
迁移方式
如果你之前一直在用 Gist，可以在 Firestore 配置区里点“从 GitHub Gist 迁移”；系统会直接读取 Gist 云端数据，有数据就写入空的 Firestore 项目，没有数据就直接连接当前 Firestore。
