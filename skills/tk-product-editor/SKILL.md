# TK Product Editor

当用户给出店小秘 TikTok 商品编辑页链接，或要求“帮我编辑店小秘商品 / 优化标题描述 / 填分类属性 / 填重量尺寸 / 上架到 TK / 发布商品”时使用这个 skill。

## 工作定位

这是采集后的商品编辑 skill。它负责把店小秘待编辑商品整理成 TikTok Shop 日本站可上架待发布产品，并在用户明确授权时提交上架。

当前用户已授权本流程：编辑完成后默认**保存并移入待发布**，即接口 `op=3` / 页面“保存并移入待发布”。这不是发布；只有用户在本次请求中明确说“上架、提交发布、发布到 TK、直接发布”，才可以使用 `op=2` 或点击最终上架/发布按钮。

## 必读资料

开始前读取：

- `references/editing-rules.md`：标题、描述、品牌、图片、变种、物流和保修规则。
- `references/selection-rules.md`：本 skill 自包含的日本站选品/合规红线。编辑前必须先过红线，违规商品停止编辑。
- `references/1688-sourcing.md`：1688 采购插件找货源、采集材质/采购价/重量尺寸和货源风险判断规则。编辑前必须完成货源核验。
- `references/sku-pricing.md`：SKU 定价规则。需要设置或调整 SKU 价格时，必须按利润计算器口径逐 SKU 生成定价报告，确认后才允许写入价格。
- `references/draft-queue-and-browser-stability.md`：店小秘 TK 采集箱队列来源、Chrome 页面通道健康检查和超时恢复规则。
- `references/api-editor-workflow.md`：店小秘接口化编辑流程。用户要求不点页面表单、走店小秘接口、脚本化读写商品对象时必须优先使用。
- `references/section-state-machine.md`：店小秘大表单分区执行模式。使用页面插件编辑时必须按区块状态机执行，不做全页扫描。

按需读取：

- `references/store-prefixes.md`：店铺名前缀规则。
- `references/category-map.md`：常见商品分类和属性映射。遇到新商品时按页面可选项判断，并在需要时更新映射。
- `references/attribute-fill-rules.md`：常见必填属性填写口径。

## 工具边界

- 店小秘商品编辑优先走接口化脚本流程；Chrome / 浏览器插件只作为备用通道，用于插件能力更新后读取当前页面、定位商品或辅助 1688 核验，不作为默认编辑通道。
- 流程必须分清两层：1688 货源核验是公共前置数据层；店小秘接口保存和店小秘页面填表只是后续写入层。无论选择哪种店小秘写入方式，都必须先完成 1688 货源核验包，除非用户明确要求跳过并接受保守估算。
- 不使用桌面控制方式代办页面点击，除非浏览器插件不可用且用户明确同意。
- 不绕过平台人工确认、付费确认、资质确认或发布确认。遇到这些状态，停止并让用户处理。
- 1688 / 采购助手出现验证码、人机验证、滑块、风险确认、空白确认或验证页时，不刷新乱撞、不换关键词硬搜，按 `references/1688-sourcing.md` 的 1688 人机验证状态机处理：记录阻塞、等用户或用户已安装插件处理、恢复后继续同一商品；超时才停止并按编辑失败回写。
- 1688 采购插件只用于找同款货源和读取商品信息；不点击 `立即下单`、`加采购车`、`跨境铺货`、`立即铺货`、`代发下单`、`加入选品池` 等会产生采购/铺货/下单影响的按钮，除非用户明确要求。
- 1688 登录状态以采购助手体系为准，不用 1688 首页、登出页或普通详情页登录状态替代判断。优先看采购助手插件面板；如果自动化无法直接读取插件 iframe，就打开采购助手点出的采购中心页面 `https://air.1688.com/kapp/1688-pc-front/procurement-center/selectionCenter/assortingPool` 做登录探针。出现 `请登录后查看最新交易信息`、`该功能仅限已登陆用户使用`、`登录` 或 `去登录` 时，立即停止 1688 核验并提醒用户先登录；用户完成登录后再继续，不用未登录插件数据继续编辑。
- 1688 货源核验不走官方开放接口；只用采购助手/1688 页面候选和已登录页面上下文读取。
- 不修改价格、库存、SKU 编码、运费模板、仓库、税务或结算相关字段，除非用户明确要求。
- 用户要求定价或改价时，只能按 `references/sku-pricing.md`：使用 1688 采购价、重量尺寸和利润计算器“定价新”口径逐 SKU 计算建议价；先出定价报告和差异报告，确认后只改报告中的 SKU 价格字段。
- 用户要求设置库存时，默认每个店小秘 SKU 填 `100`；只改库存字段和必要的商品级库存汇总字段，不改 SKU 编码、仓库、运费模板、税务或结算字段。
- 不虚构商品材质、认证、功效、适用范围、套装数量、品牌或赠品。

## 待编辑队列来源

- 待编辑商品入口以店小秘 `TikTok > 采集箱` 当前页面为准，或以用户明确给出的店小秘编辑页链接为准。
- 不把 `collection_records` / 采集表里的 `店小秘编辑状态=未编辑` 当作待编辑队列；采集表可能包含用户尚未审核或尚未转移到 TK 采集箱的商品。
- 采集表只用于编辑前辅助匹配商品名称、商品链接或商品 ID，以及保存成功后的回写；不得从采集表挑商品绕过店小秘采集箱。
- 如果用户没有指定具体商品，从 TK 采集箱当前列表的可见商品开始逐条处理；明显命中红线的商品可以跳过并记录决定性原因，再处理下一条。

## 店小秘页面操作约束

- 用户要求“接口/脚本化编辑”“不点表单”“走页面自身接口”时，优先按 `references/api-editor-workflow.md` 执行：输入编辑链接，调用读取接口，生成差异报告，完成编辑后按当前授权保存并移入待发布。
- 接口化编辑默认不依赖 Codex Chrome 插件；插件路线保留为备用，只有接口登录态不可用、用户明确要求或插件能力更新后才尝试。
- 接口化保存主路是“真实编辑页 Pinia store 改值 + 页面前端保存命令 + `edit.json` 验证”：从页面资源动态识别 TikTok 编辑 store，优先复用 live store；需要改价/库存时修改 `tiktokGlobalSkuDataStore` 内 `originalPrice`、`availableStock`、`stockInfos.available_stock`，先用 `getFormData()` 预览保存 DTO，再触发页面已有保存命令提交 `op=1`。
- 不要把 `/api/tiktokProduct/edit.json` 的原始读取对象直接当成保存对象上传。能拿到分区 `getFormData()` / 保存 service 时可合并前端 DTO；拿不到组件 `ref` 时按 `references/api-editor-workflow.md` 的 Pinia store 路线执行。
- 如果 `/api/tiktokProduct/add.json` 返回 `保存对象转换异常,请联系我们!`，通常说明保存对象形状不对；立即停止原始对象上传路线，不要重复撞，切换到 Pinia store / 前端保存路线，未重新读取验证成功前不得回写采集表。
- 描述保存后必须用 `edit.json` 验证，不能只看页面编辑器或 textarea。若描述为空，按 `references/api-editor-workflow.md` 的 zip DTO 路线补根字段 `description` 后 `op=1` 保存。
- 接口化编辑保存前必须有差异报告；不得直接保存。保存只允许 `op=1`，除非用户在当前步骤明确授权其他动作。
- 接口化编辑默认只能改分类、品牌、属性、标题、描述、重量尺寸等允许字段；必须保留 SKU、价格、库存、仓库、运费模板、税务和结算相关字段，除非用户在本次任务明确授权改价或改库存，并已生成对应差异报告。
- 接口化编辑不能只依赖 `service.D(categoryId)` 判断产品属性；该接口可能只返回变种属性。必须从页面 Pinia `tiktokGlobalAddStore.attrsInfo.attrsList` 读取完整产品属性，凡 `isMandatory=1` 的属性都要按 1688 货源或商品事实填写。日本站常见必填包括 `原産国/原产地` 和 `梱包材/材质`，保存后必须用 `service.b(productId)` 或 `edit.json` 验证 `productAttributes` 里确实存在对应属性。
- 接口化编辑遇到登录失效、验证、接口异常、非 JSON 或业务错误时停止，不做规避。
- 页面编辑默认使用 `references/section-state-machine.md` 的分区执行模式：先准备编辑包，再按基本信息、产品属性、产品信息、图片、变种/SKU、物流保修、保存回写逐区推进。
- 店小秘 TikTok 编辑页在选择分类后会展开产品属性、变种和 SKU 表格。不要在分类后做全页 DOM 快照、`body.innerText`、全量 `input` 扫描、整页截图、可见 DOM 树、坐标元素探针或宽泛 locator 过滤；这些动作容易让页面动作通道排队超时。
- 每一步只处理一个区块：先点左侧锚点跳到目标区块，再用字段名称附近的唯一控件、已有 `value`、`name`、`placeholder` 或弹窗内局部 locator 精准定位。
- 每个区块开始前先确认进入条件，结束前必须做完成校验；如果校验失败，停在当前区块恢复，不回到页面顶部重跑。
- 每完成一个区块都记录区块状态：`pending`、`done`、`blocked` 或 `skipped`，并记录关键填写值，方便中断后从最近区块继续。
- 类目弹窗优先使用分类搜索，选中末级类目后确认。确认按钮如果与页面其他“选择”按钮重名，必须限定在弹窗内或换用更明确的局部定位。
- 分类后先确认已选类目、必填属性和变种主题是否出现；不要因为页面卡顿重复点击分类或保存按钮。
- 富文本描述优先使用编辑器源码模式或编辑器 iframe 的 `body`，只替换描述正文，不点击 AI 生成。
- 失败恢复优先采用当前区块内关闭弹窗、重新点左侧锚点或重新接管同一标签页；不要为同一商品重复打开新店小秘页。
- 一个店小秘页面动作超时后，先停止继续发送页面动作；只允许做无副作用的标签页标题/URL 检查。按 `draft-queue-and-browser-stability.md` 恢复当前区块，不能连续重试同类页面动作。
- 试跑或用户未明确授权时，只允许填写草稿字段，不点击 `保存`、`保存并移入待发布` 或 `发布`。当前线程已明确授权“编辑完的商品保存并转入待发布”，所以后续完成编辑且验证无误后使用 `op=3`；仍然禁止 `op=2` 发布。

## 采集表回写

- 编辑前先从页面或采集表确认目标账号、商品链接/商品 ID/商品键和商品名称，确保能对应到 Firestore `collection_records` 里的既有商品。
- 编辑技能只处理编辑字段，字段顺序固定为：`店小秘编辑状态`、`编辑时间`、`编辑标题`、`编辑判断`。采集字段 `账号`、`选品分`、`商品名称`、`店铺名`、`商品价格`、`商品近7天销量`、`采集时间`、`采集状态`、`选品判断` 归采集技能维护，编辑技能不得修改。
- 未回写前页面可显示为 `未编辑`，但采集技能不写编辑字段。编辑完成并保存到待发布后，回写同一条采集表记录：`店小秘编辑状态=已编辑`、`编辑时间=<保存验证成功时间>`、`编辑标题=<最终日语标题>`、`编辑判断=<成功判断>`。
- 编辑阶段判定不合格、跳过或停止时，回写同一条采集表记录：`店小秘编辑状态=编辑失败`、`编辑时间=<判断时间>`、`编辑判断=<失败的第一决定性原因>`；不写 `编辑标题`，不写 `编辑失败原因`、`风险标记`，也不得修改 `采集状态` 或 `选品判断`。
- 回写优先使用项目/采集 skill 的命令：`node <tk-product-selection-skill-dir>/scripts/firestore-sync.mjs mark-dxm-edited --account <账号名> --product-url <商品链接> --product-name <商品名称> --edited-title <编辑标题> --judgement <成功判断>`。
- 不合格回写优先使用项目/采集 skill 的命令：`node <tk-product-selection-skill-dir>/scripts/firestore-sync.mjs mark-dxm-rejected --account <账号名> --product-url <商品链接> --product-name <商品名称> --reason <决定性原因>`。
- 该命令只能更新数据库中已经存在的采集记录。找不到对应商品或匹配到多条时停止，不新增采集记录，不把旧记录算成本次新编辑。

## 1688 货源核验

- 编辑前必须用 1688 采购插件或 1688 同款页面核验货源；优先从店小秘商品图触发插件找货源。这一步是店小秘接口保存和页面填表的共同前置，不属于任一写入路线。
- 1688 当前默认路线：店小秘商品图/标题/变种 -> 采购助手以图搜同款或 1688 页面搜索 -> 取 1-3 个候选 -> 打开候选详情页 -> 在已登录 Chrome 页面上下文结构化读取页面数据和 DOM -> 生成货源核验包。
- 货源核验开始时先确认 1688 采购助手已登录；优先看插件面板，无法读取插件面板时用采购中心选品池页面做登录探针。未登录时先提醒用户登录，不继续采集材质、价格、销量、交易或物流数据。
- 1688 图搜、搜索结果、候选详情页或 SKU 价格读取阶段触发人机验证时，立即进入 1688 人机验证状态机。不得把验证页、空白页、旧缓存、错误主体图搜结果或关键词误搜结果当成货源核验结果。
- 每个商品只保留必要的 1688 候选页，通常查看 1-3 个候选；提取完立即关闭无用候选页，避免页面越开越多。
- 核验字段至少包括：1688 链接、供应商、采购价/变种价、起批量、库存、材质、规格/尺寸、重量、发货地、运费/发货时效、评价或差评风险。
- 货源数据只能作为上架编辑依据，不写进面向买家的标题或描述里的价格、优惠、销量、供应商名。
- 材质、重量、尺寸和变种必须优先按可信 1688 货源填写；货源字段缺失、冲突或同款不确定时，不要编造，改为保守估算或停止确认。
- 需要改价时，1688 货源核验包必须额外提供每个目标 SKU 的采购价、重量尺寸和货物类型，用于生成 SKU 定价包。

## 默认流程

1. 从用户给出的店小秘编辑页或店小秘 `TikTok > 采集箱` 当前列表取得目标商品；确认是 TikTok 商品编辑页，并确认当前商品 ID、店铺名和商品名称。
2. 建立本次编辑包：读取当前商品必要信息、图片、变种、商品来源信息和已填字段；用 1688 采购插件找同款货源，按 `references/1688-sourcing.md` 采集货源数据并判断可信度。需要改价时，再按 `references/sku-pricing.md` 生成逐 SKU 定价包。
3. 按 `references/selection-rules.md` 做上架前红线复核；如果命中禁售、暂不支持、跨境禁运、运营红线或货源风险，停止编辑并说明原因。
4. 在编辑包里提前生成分类判断、必填属性候选、标题、描述、图片处理决策、变种映射、重量尺寸估算和数据库回写键；不要进入页面后一边想一边填。
5. 按 `references/section-state-machine.md` 分区执行。每个区块只处理本区块字段，并在通过完成校验后才进入下一区块。
6. 基本信息区：选择新版产品分类，等待属性刷新，选择 `无品牌` 或页面对应无品牌选项，确认产地等必要基础字段。
7. 产品属性区：填写所有必填属性；优先按可信 1688 货源和商品真实信息填写，不确定时使用安全中性值或停止让用户确认。
   - 接口化保存时先读 `tiktokGlobalAddStore.attrsInfo.attrsList`，用其中的 `attrId/attrName/valueList/isMandatory` 生成属性 JSON；不要把 `productAttributes` 清空保存。原产地按可信货源发货/生产信息填写，材质按 1688 明确材质映射到平台枚举。
8. 产品信息区：写入优化后的日语标题和描述。描述必须保留原有图片和 HTML 结构；有原文字就优化成日文写回，没有文字才在图片后追加简短日文说明。标题开头必须带店铺前缀，不写包邮、送料無料、官方、大牌、疗效、最强等不实或高风险词。
9. 图片区：检查主图是否清晰真实；如果首图差，生成不虚构实物的白底/简洁主图，并上传替换或追加到合适位置。
10. 变种/SKU 区：按店小秘已有变种和可信 1688 货源 SKU 对齐填写变种属性；不新建无依据变种，不修改 SKU 编码。价格只有在用户要求并确认 SKU 定价报告后才修改；库存只有在用户要求时修改，默认每个 SKU 填 `100`。
11. 物流和保修区：优先使用可信 1688 包装信息里的重量/尺寸；缺少完整尺寸时按实物形态加包装冗余保守估算，不低报。
12. 保存与回写区：保存并移入待发布（接口 `op=3`）。只有用户明确授权发布时，再做最终复核并提交上架。保存成功并验证 `dxmOfflineState=waitPublish` 后按“采集表回写”更新数据库；如果未保存成功，不标记 `店小秘编辑状态=已编辑`。

## 必须停止的情况

- 商品命中选品/合规红线。
- 页面要求平台人工确认、付费确认、资质证明或发布确认。
- 1688 采购助手插件面板或采购中心选品池登录探针处于未登录状态，需要用户先登录。
- 1688 图搜、搜索结果、详情页或 SKU 价格读取出现人机验证且等待超时，无法恢复同一商品的货源核验。
- 必填属性无法根据商品真实信息判断。
- 找不到可信 1688 货源，或货源和店小秘商品在外观、规格、材质、套装数量上明显不一致。
- 1688 货源存在严重差评、货不对板、无法提供必要资料、库存/价格异常或发货能力明显不稳。
- 用户要求改价但无法把店小秘 SKU 与可信 1688 SKU 一一对齐，或缺少采购价、重量尺寸导致无法按利润计算器生成定价报告。
- 商品图片不足以确认实物，且生成图片会导致实物不一致。
- 店铺名、目标账号或发布站点不明确。
- 最终发布前发现标题、描述、图片、分类、属性、变种或物流信息存在不确定风险。

## 输出要求

完成后简短汇报：

- 商品编辑状态：已移入待发布 / 已提交上架 / 已停止。
- 分类和关键属性。
- 1688 货源核验：选中的 1688 链接、供应商、采购价、材质、重量尺寸、风险备注。
- SKU 定价：是否按利润计算器生成/写入价格，关键参数和异常 SKU。
- 标题前缀和最终标题。
- 描述是否已优化。
- 图片是否调整，若生成主图说明生成原因。
- 重量尺寸填写值。
- 采集表回写：是否已回写 `店小秘编辑状态`、`编辑时间`、`编辑标题` 和 `编辑判断`。
- 未处理或需要用户确认的项目。

如果停止，说明第一个决定性原因，不列泛泛风险。
