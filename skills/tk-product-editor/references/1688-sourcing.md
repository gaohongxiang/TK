# 1688 货源核验规则

## 目标

在编辑店小秘商品前，用 1688 采购插件或 1688 详情页确认商品真实货源信息，减少材质、规格、重量尺寸和变种误填。

核验结果用于：

- 判断这个商品是否值得继续编辑上架。
- 填写店小秘产品属性、变种、标题描述中的真实规格信息。
- 填写物流重量尺寸。
- 后续沉淀到商品管理的 1688 链接、采购价和物流参数。

## 入口

- 优先在店小秘商品详情页使用已安装的 1688 采购插件，按商品主图/详情图找同款。
- 也可以打开插件返回的 1688 候选商品页；URL 常见来源参数如 `fromkv=cbuPcPlugin:imageSearchDrawerCard`。
- 开始核验时先确认 1688 采购助手已登录，以采购助手体系里的状态为准，不用 1688 首页、登出页或普通详情页登录状态替代判断。
- 优先看采购助手插件面板；如果自动化不能直接读取插件 iframe，就打开采购助手点出的采购中心选品池页面 `https://air.1688.com/kapp/1688-pc-front/procurement-center/selectionCenter/assortingPool` 作为登录探针。
- 插件卡片、插件 iframe、插件弹窗或采购中心页面出现 `请登录后查看最新交易信息`、`该功能仅限已登陆用户使用`、`登录`、`去登录`、登录弹窗、验证码等状态时，停止并提醒用户先登录，等用户完成登录后再继续。
- 不使用桌面控制方式点击插件，除非 Chrome 插件路径不可用且用户明确同意。

## 当前默认路线

1688 货源核验是店小秘接口保存和页面填表的共同前置数据层。不走 1688 官方开放接口。

默认流程：

1. 从店小秘商品图、标题和变种信息出发。
2. 必须先用 1688 采购助手或 1688 页面“以图搜款”找同款；首选店小秘当前商品首张主图，首图空结果或明显误匹配时，再按店小秘主图顺序、SKU 图、详情图逐张顺延，不把首图空结果当作找货失败。
   - Web 端图搜前必须清空旧关键词、类目筛选、价格筛选等搜索状态；优先上传“只包含商品主体”的裁剪图或店小秘原图，不要上传带大面积留白、页面水印、拼图、截图 UI 的图片。上传后必须确认页面显示已上传图片，并检查前 5-10 个可见结果是否有同图/近同图，不能只取第一个结果。
   - 1688 图搜识别到多个主体时，必须按店小秘当前售卖商品选择主体框；例如卖包就框选包本体，不框选手、手镯、模特身体、背景、吊牌以外的饰品或页面文字。结果中出现手镯、衣服、人体局部等明显不是目标商品的候选，视为主体框选错误或候选过滤失败，必须重新框选/裁剪后再搜。
   - 框选主体时要覆盖目标商品的关键外观特征：整体轮廓、主材质、肩带/提手、明显 logo/吊带、开合结构和正面装饰；不要只框局部 logo、局部肩带或配件。
   - 用户提供的手机图搜截图、人工图搜结果或采购助手返回结果优先作为纠偏线索；不要假设其一定来自 1688 App 或某个固定客户端。如果用户截图里已经出现同图候选、标题或供应商名，必须用这些线索反向定位 1688 详情页并核验，不能继续按 Web 图搜的误匹配结果判定“找不到同款”。
   - 如果 Web 图搜第一页结果与目标商品明显不一致，而用户人工图搜能找到同款，判定为当前 Web 图搜链路污染或识别失败；换裁剪图、换无关键词图搜入口、或用截图候选标题/供应商反查，不得直接回填“不合格”。
   - 文字关键词搜索只能作为图搜后的补充：用于验证候选、补找缺色/缺规格、或在图搜入口不可用/触发验证后记录 fallback。不得跳过图搜直接用关键词结果判断同款。
   - 1688 文字搜索 URL 的 `keywords` 必须按 GBK/GB18030 编码后再拼接，不能用默认 UTF-8 URL 编码；UTF-8 关键词会在 1688 搜索页显示乱码，并可能返回完全不相关结果。发现搜索标题或结果乱码时，立即丢弃该结果，重新用 GBK 编码或回到以图搜。
3. 只取 1-3 个最像的候选货源。
4. 打开候选 1688 详情页。
5. 在已登录 Chrome 页面上下文里结构化读取页面内嵌 JSON、页面接口响应和公开 DOM。
6. 按字段可信度生成货源核验包，再进入店小秘编辑包。

执行边界：

- 不读取 Chrome cookie 文件，不绕过验证码、风控、扫码或登录验证。
- 如果页面出现 `bxpunish`、验证码、登录失效或人机验证，按下面的 1688 人机验证状态机处理；不要刷新乱撞、换关键词硬搜或把验证页当结果。
- 采购助手只用于以图搜同款和给出候选链接；候选详情页的信息仍按结构化字段提取和可信度判断，不把插件卡片文案直接当最终依据。
- 不使用 1688 官方开放接口作为货源核验输入。

不要用服务器裸 `curl` 抓 1688 大市场搜索或详情页作为默认路线。实测 1688 搜索/详情页会返回 `bxpunish` / 验证页；自动绕过风控不属于本流程。

## 禁止动作

不要点击这些会造成采购、铺货、加入清单或下单影响的按钮：

- `立即下单`
- `加采购车`
- `跨境铺货`
- `立即铺货`
- `加铺货单`
- `代发下单`
- `加入选品池`
- 任何付款、授权、绑定、确认采购、确认铺货按钮

遇到登录、验证码、付费确认、授权确认、平台人工确认时停止，让用户处理。采购助手未登录时不要继续读取价格、交易、销量、物流或供应商评分等可能缺失/失真的数据，也不要只凭 1688 网页登录态判定插件可用。

## 1688 人机验证状态机

1688 人机验证包括采购助手、1688 以图搜、搜索结果页、候选详情页或 SKU 价格/运费读取过程中出现验证码、滑块、安全验证、风险确认、空白确认页、`bxpunish`、`login.taobao.com` 验证跳转，或页面提示 `请完成验证` / `安全验证` / `拖动滑块` / `验证码`。

发现后：

1. 立即停止当前 1688 页面操作；不要继续点击、刷新、换关键词、读取大段 DOM、重复上传图片或把旧结果当成当前结果。
2. 在当前运行目录写入或更新 `page_confirmation_state.json`。没有运行目录时，写到当前商品的临时工作目录，例如 `data/dxm-api/<店小秘商品ID>/page_confirmation_state.json`。只记录恢复需要的信息：`status: "verification_blocked"`、`platform: "1688"`、账号/店铺、阶段、店小秘商品 ID、商品名称、店小秘链接、当前 1688 URL、已选候选 URL、阻塞时间、最近检查时间；不记录账号资料、cookie、LocalStorage、截图或长页面文本。
3. 告知用户“1688 页面需要人工确认，处理后我继续同一个商品”。如果用户已安装自动确认插件，保持当前页面，轻量检查 URL、标题和少量页面标记即可，不反复催问。
4. 每 10-15 秒轻量检查一次，最多等待 5 分钟。轻量检查只判断验证层是否消失、URL 是否回到搜索/详情页、目标商品主体或候选信息是否重新可读。
5. 页面恢复后，把 `page_confirmation_state.json` 更新为 `status: "resumed"`，继续同一个商品、同一张图或同一个候选详情页；不要从头换路线，也不要因为触发过验证就改用关键词误搜。
6. 等待超时或页面仍无法恢复时，把状态更新为 `status: "timeout"`，关闭多余的 1688 候选页，停止编辑当前商品，不保存店小秘；回写同一条采集表记录：`店小秘编辑状态=编辑失败`、`编辑时间=<当前时间>`、`编辑判断=1688 页面出现人机验证，等待超时，未完成货源核验。`，`编辑标题` 留空。

验证期间不能把空白页、遮罩层、半页数据、旧缓存、错误主体框选结果或乱码关键词结果当作真实货源；也不能把未完成 1688 货源核验的商品标记为已编辑。

## 候选货源选择

通常查看 1-3 个候选即可。优先选择：

- 主图、细节图、颜色/规格和店小秘商品高度一致。
- 有明确 SKU 价格、库存、材质、规格、重量或包装信息。
- 起批量低，支持代发或跨境贴标/箱唛。
- 店铺回头率、发货率、好评率相对正常。
- 差评少，且没有集中提到货不对板、质量差、缺件、售后资料拖延。

颜色匹配口径：

- 1688 颜色名称不要求和店小秘 SKU 文案完全一致；以商品图和实物颜色为准做映射，例如 `パープル`/`紫`、`ブルー`/`蓝`、`赤色`/`红色` 可以互相映射。
- 同一货源缺少个别颜色时，如果商品款式、材质、规格、本数和包装一致，可以用同规格同本数 SKU 的采购成本临时核价，并在货源核验包里标注“缺色估算”；后续采购时再补更精确货源。
- 不允许跨规格估算：4 管不能按 6 管成本反推，普通款不能按织带款、加厚款、套装款反推，除非用户明确接受并在报告中标注。
- 如果缺色同时伴随图片款式、管数、材质或尺寸差异，不能估算，必须继续找货源或停止确认。

降权或停止：

- 图片相似但套装数量、配件、材质、尺寸明显不同。
- 评价里出现货不对板、质量严重不稳定、售后拒绝配合、发票/合同/证明长期拖延。
- 材质写贵金属、天然石、玻璃、陶瓷、食品接触、儿童/婴儿、医疗功效等高风险信息但无证明。
- 起批量、价格或库存异常，无法判断实际成交规格。
- 货源本身属于选品红线或 TikTok 日本站禁限售/暂不支持品。

## 必采字段

从选中的 1688 货源记录：

- `1688 链接`
- `供应商`
- `1688 商品标题`
- `采购价`：优先按店小秘/TK 采集箱目标 SKU 的实际售卖数量或套装数量，在 1688 详情页选择对应 SKU 和数量后，记录页面展示的商品金额、预估运费和合计成本；有多个变种时逐 SKU 记录。单件 SKU 才用数量 `1`，多件套、组合装或 `2個セット` 等必须按实际件数读取。不要只看搜索页运费或详情页 SKU 标价。
- `起批量`
- `库存`
- `材质`
- `规格/尺寸`
- `重量`
- `发货地`
- `运费/发货时效`
- `评价/差评风险`
- `选用理由` 或 `不用理由`

价格、销量、供应商名只用于内部采购和核算，不写进 TikTok 买家可见标题/描述。

## 字段可信度

高可信：

- 1688 商品属性里的 `材质`、`规格`。
- 1688 `包装信息` 或 `商品件重尺` 里的重量/尺寸。
- SKU 行里的目标颜色/规格价格、库存。

中可信：

- 详情图里的尺寸示意。
- 标题里的规格词。
- 插件侧栏的统计字段。

低可信：

- 推荐商品区的信息。
- 评价里非卖家发布的规格描述。
- 自动生成标题、AI 标题、营销文案。

字段冲突时，以商品属性和包装信息优先；仍冲突就停止确认，不要凭感觉编。

## 重量尺寸口径

- 1688 只给重量时，店小秘重量按实重加包装冗余填写，通常上浮 10%-30%，最低也要覆盖外包装。
- 1688 给完整 `长×宽×高` 时，按包装后尺寸填写；如果看起来是实物尺寸，另加合理包装冗余。
- 1688 只给两段规格，如 `38*25`，不能直接当完整尺寸；可按实物形态补一个保守厚度，并在汇报里说明。
- 多 SKU 重量不同，按目标 SKU 分别记录；店小秘只能填统一重量时，用最重目标 SKU 加包装冗余。
- 不低报重量和尺寸。长边超过 35-40cm、重量超过 1kg 要强复核；超过 2kg 或不可压缩默认停止。

## 和店小秘编辑的关系

- 分类、材质、规格、变种、标题描述和物流参数都优先引用可信 1688 货源。
- 不把 1688 采购价、供应商、起批量、销量、优惠写给 TikTok 买家。
- 生成主图时，只能基于真实货源和店小秘现有商品图做干净表达，不能虚构套装、配件或材质。
- 保存草稿后，汇报中必须列出选中的 1688 货源、采购价、材质、重量尺寸和风险备注。
