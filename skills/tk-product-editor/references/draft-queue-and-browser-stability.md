# 店小秘采集箱与浏览器稳定策略

## 队列来源

- 待编辑商品只来自店小秘 `TikTok > 采集箱` 当前列表，或用户明确给出的店小秘 TikTok 编辑页链接。
- 不把采集表 `collection_records` 当作待编辑队列。采集表里的商品可能尚未经过用户审核，或尚未被用户转移进 TK 采集箱。
- 采集表只用于辅助匹配和保存后回写：目标账号、商品链接、商品 ID、商品名称用于匹配；保存后只回写 `店小秘编辑状态`、`编辑时间`、`编辑标题`、`编辑判断`。
- 使用接口取采集箱列表时必须显式传 `dxmState=draft`。不要用无参 `/api/tiktokProduct/pageList.json` 结果当待编辑队列；无参或错误参数可能混入在线产品。
- 从采集箱列表或编辑页拿到商品后，再读 `/api/tiktokProduct/edit.json?id=<商品ID>` 或 `service.b(productId)` 二次确认 `dxmState === 'draft'`。若为 `online` 或其他状态，立即停止并跳过，不保存、不回写。
- 已验证采集箱列表参数口径：`shopId=-1`、`pageNo=1`、`pageSize=50`、`searchType=0`、`searchValue=`、`dxmState=draft`、`sortName=0`、`sortValue=0`。只从该响应里的 `page.list` 取候选。

## 浏览器操作优先级

1. 使用 Chrome 插件的短请求确认操作窗口：列标签、标题、URL、当前页是否为店小秘采集箱或编辑页。
2. 在采集箱列表只取当前可见行的必要信息，不做整页文本、截图或按钮数量统计。
3. 进入编辑页后按 `section-state-machine.md` 的区块处理；每次只操作一个区块里的字段。
4. 对产品信息区优先使用人工等价最小链路：点字段、全选、粘贴、轻量校验。
5. 页面动作失败时停在当前区块恢复；不要重开同一商品多个编辑页。

## 禁止的高风险观察动作

在店小秘采集箱和编辑页里不要使用：

- `body.innerText` 或整页文本读取。
- 全页 DOM snapshot、可见 DOM 树、全量 input/textarea/select 扫描。
- 整页截图、坐标元素探针、按钮数量统计。
- 宽泛 `getByText('编辑')`、全页 locator 过滤或多候选逐个试探。
- 给点击、输入、剪贴板动作外包 `Promise.race` 做假超时。

这些动作可能不会取消已发出的浏览器动作，后续动作会排队，导致同一标签页持续超时。

## 健康检查

开始页面动作前只做轻量检查：

- 能列出 Chrome 标签页。
- 能看到店小秘采集箱或编辑页的标题和 URL。
- 当前商品只保留一个可操作编辑页；发现重复编辑页时优先使用当前会话里的标签，不重复打开。

不要用店小秘页面内 `evaluate`、可见 DOM 或截图作为健康检查。需要确认页面内容时，先通过采集箱当前可见行或编辑区块局部控件完成。

## 超时恢复

一次店小秘页面动作超时后：

1. 立即停止发送新的页面动作。
2. 只做无副作用的标签页标题/URL 检查，确认目标标签还在。
3. 如果短请求可用，重新接管同一标签，并回到当前区块锚点。
4. 如果同一类型页面动作再次超时，不要连续第三次重试；改用另一种低成本路径，例如键盘最小链路、关闭弹窗、刷新当前页后从当前区块续跑。
5. 如果点击、键盘、局部字段定位都对同一标签超时，判定该标签页面动作通道不可用；停止并让用户选择刷新登录窗口、重启 Chrome 插件会话或明确授权桌面点击。

接口化路线补充：

- 店小秘接口化保存不依赖 Codex Chrome 插件页面动作；可用真实 Chrome 已登录标签页的页面上下文执行 JS。
- Chrome AppleScript 执行 JS 时，优先使用 `tell tab N of window 1` 后 `execute javascript js` 的写法；不要写成 `execute javascript js in tab N of window 1`，部分 Chrome 版本会把它解析失败。
- JS 脚本里含日文/中文时，用 base64/JSON 在页面上下文解码，避免 AppleScript 文件读取造成乱码。

## 采集箱取商品策略

- 用户没有指定商品时，从当前采集箱可见列表第一条未处理商品开始。
- 列表中明显命中红线的商品可以跳过，例如宠物垫、大垫、贴纸纸品、汽车配件、尖锐工具、玩具人偶、首饰钥匙扣等；记录第一个决定性原因。
- 打开编辑页后再确认商品名称、店铺账号、商品 ID 和商品来源。采集表匹配失败不影响页面编辑，但保存成功后不能回写 `店小秘编辑状态=已编辑`。

## 何时允许桌面点击

- 默认仍优先 Chrome 插件。
- 只有在 Chrome 插件短请求可用、但同一店小秘标签的页面动作连续失败，且用户明确同意“桌面点击方式”时，才改用桌面控制。
- 桌面点击仍遵守同样边界：不发布、不采购、不下单、不接受付费/资质/平台确认，不绕过验证码。
