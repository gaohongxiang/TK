# 店小秘 TikTok 商品编辑规则

## 标题

- 标题必须使用日语，开头必须带店铺前缀，例如 `NOMA-`。
- 标题应包含商品核心词、主要用途、关键规格或适用场景。
- 不写 `送料無料`、`包邮`、`公式`、`正規品`、`No.1`、`最強`、医疗/治疗/减肥等高风险词。
- 不写品牌、IP、角色名或兼容品牌词，除非用户明确提供授权依据。
- 不堆砌无关关键词，不承诺页面无法证明的材质、数量、认证、效果。

## 描述

描述使用日语；如果店小秘原描述里已有图片或 HTML 内容，必须先保留原有图片和非文本结构，再只替换或补充文字描述。

- 原描述有 `<img>` 时，不删除、不重排、不替换图片；图片说明图已经能表达的信息不用重复处理。
- 原描述有文字时，提取文字含义，优化成自然、合规的日文版后写回描述。
- 原描述只有图片、没有可用文字时，可以在图片后追加简短日文商品说明；不要因为“有图片”就让文字描述为空。
- 不用空白模板覆盖描述，不把描述改成只剩图片，除非原商品本来没有可用文字且用户明确要求只保留图片。

文字描述建议结构：

1. 商品用途和适合场景。
2. 主要特点，保持真实克制。
3. 规格/数量/颜色说明，按页面已有信息写，不确定不写死。
4. 使用注意事项。

不要写：

- 医疗、治疗、瘦身、增高、改善疾病等功效。
- 绝对化承诺，如永久、100%、必ず、完全防止。
- 包邮、价格、促销、站外联系方式。
- 品牌/IP/授权暗示。

## 品牌

- 默认选择 `无品牌`、`No Brand`、`ノーブランド` 或页面同义选项。
- 如果页面必须填写品牌文本，填页面允许的无品牌值。
- 不用店铺名冒充品牌。

## 分类

- 先按商品真实用途选择最低风险分类。
- 不把高风险商品塞进低风险类目规避审核。
- 有可信 1688 货源时，分类判断要参考 1688 商品属性、产品类别和真实用途。
- 选择分类后必须等待产品属性和变种属性刷新，再继续填写。
- 如果分类候选明显不匹配，停止让用户确认。
- 分类选择后页面会展开大量属性和变种字段。后续操作禁止全页扫描输入框，应按区块锚点和字段唯一特征逐项处理。

## 图片

合格主图标准：

- 商品主体清晰，占画面主体。
- 不遮挡、不模糊、不带明显水印、无多余促销文案。
- 与实际售卖商品一致，不夸大数量、材质、尺寸、品牌或场景。

需要生成主图时：

- 只能基于真实商品图做干净白底、轻度构图优化或突出实物。
- 不能虚构套装、配件、品牌、功能效果、尺寸对比或使用场景。
- 不生成看起来像官方品牌图、医疗/功效图、夸张前后对比图。

## 变种

- 只填写页面已有或商品真实可判断的颜色、尺寸、数量、规格。
- 店小秘已有变种必须和可信 1688 目标 SKU 对齐；采购价、库存和重量不同的 SKU 要分别记录，不要混成一个规格。
- 不凭空新增 SKU。
- 颜色用日语常见色名，例如 ブラック、ホワイト、グレー、ピンク。
- 尺寸不确定时不要填写具体厘米值，先查页面已有信息或停止确认。
- 不修改 SKU 编码、库存和发货时效，除非用户明确要求。
- 用户要求设置库存时，默认每个店小秘 SKU 填 `100`；只改库存字段和必要的商品级库存汇总字段。
- 价格默认不改；用户要求定价或改价时，必须按 `sku-pricing.md` 使用 1688 采购价、重量尺寸和利润计算器“定价新”口径生成逐 SKU 定价报告，确认后只改报告中的 SKU 价格字段。

## 物流和保修

- 重量和尺寸优先使用可信 1688 包装信息或商品件重尺；缺失时才保守估算，不低报。
- 小件轻货可按实物体积加包装估算；不可压缩、易压坏或有外盒商品要加包装冗余。
- 1688 只给两段规格时，不能直接当完整包装尺寸；补厚度或包装冗余时必须在汇报里说明是估算。
- 多 SKU 重量不同时，店小秘只能填统一重量就按最重目标 SKU 加包装冗余。
- 保修字段选择平台和类目允许的安全默认值，不承诺额外保修。

## 1688 货源

- 编辑前必须完成 1688 货源核验，除非用户明确说跳过且接受字段只能保守估算。
- 如果 1688 采购助手插件面板未登录，或采购中心选品池登录探针显示未登录，先提醒用户登录，登录完成前不继续货源核验和店小秘编辑；不要用 1688 首页、登出页或普通详情页登录状态替代采购助手登录状态。
- 1688 信息只用于内部采购核验、属性填写和物流估算，不写入买家可见的价格、销量、供应商或优惠。
- 货源和店小秘商品不一致时，优先停止，不要强行编辑。

## 发布

- 默认保存草稿。
- 只有用户明确要求本次直接上架/提交发布时，才可以点击最终发布按钮。
- 发布前必须复核分类、必填属性、品牌、标题、描述、图片、变种、重量尺寸。

## 采集表回写

- 保存草稿成功后，回写同一条数据库采集记录，不新增商品。
- 匹配优先级：目标账号 + 商品链接，其次商品 ID/商品键，最后才用商品名称精确匹配。
- 编辑技能只回写编辑字段，字段顺序固定为：`店小秘编辑状态`、`编辑时间`、`编辑标题`、`编辑判断`。
- 未回写前页面可显示 `店小秘编辑状态=未编辑`，但采集技能不初始化或写入编辑字段；编辑成功写 `已编辑`，编辑失败/跳过/不合格写 `编辑失败`。
- 编辑成功时 `编辑时间` 写保存验证成功时间，`编辑标题` 写最终日语标题，`编辑判断` 写成功判断。
- 编辑失败、跳过或判定不合格时，只写 `店小秘编辑状态=编辑失败`、`编辑时间`、`编辑判断=<第一决定性原因>`；不写 `编辑标题`、`编辑失败原因` 或 `风险标记`，不得修改原有 `采集状态` 或 `选品判断`。
- 找不到或匹配多条采集记录时停止，让用户补充商品链接或商品 ID。
