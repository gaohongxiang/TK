# 店小秘分区执行模式

店小秘 TikTok 编辑页是重表单。使用 Chrome / 浏览器插件编辑时，默认按本状态机执行，避免全页 DOM 扫描、重复开页和跨区误改。

## 总原则

- 待编辑商品以店小秘 `TikTok > 采集箱` 当前列表或用户给出的店小秘编辑页为准；采集表不是待编辑队列。
- 开始页面动作前先做轻量窗口检查：只列标签、标题和 URL，不用整页文本、DOM、截图或元素探针做健康检查。
- 先准备编辑包，再进入页面填写。编辑包包含：商品 ID、店铺名、商品名称、目标分类、必填属性候选、品牌值、日语标题、日语描述、图片决策、变种映射、重量尺寸、1688 货源依据、数据库回写键。
- 每次只处理一个区块。进入区块后只定位本区块可见字段，不读取全页所有输入框，不按宽泛文本反复过滤。
- 每个区块都有状态：`pending`、`done`、`blocked`、`skipped`。区块完成后记录关键填写值和校验结果。
- 区块失败时停在当前区块恢复，不从页面顶部重跑；优先关闭弹窗、重新点当前锚点、重新读取当前区块可见 DOM 或重新接管同一标签页。
- 同一商品只保留一个店小秘编辑页。1688 候选页提取完立即关闭无用页，通常只保留最终可信货源页。
- 未明确授权时只允许填草稿字段，不点击 `保存`、`保存并移入待发布`、`发布`、`更新`。
- 不要用 `Promise.race` 或外层计时器给 Chrome 输入动作、点击动作、剪贴板动作做“假超时”。这不会取消已经发给浏览器插件的动作，反而会让后续动作排队，造成越来越多的超时。
- 一个 Chrome 动作超时后，先停止继续发送新动作；只做无副作用的标签页列表/URL 检查。确认当前动作通道恢复，或重建浏览器会话、刷新当前页后，再从当前区块续跑。
- 产品信息区优先使用人工等价的最小动作链：点标题输入框、全选、粘贴标题；点描述编辑器或源码模式、全选、粘贴描述。只有最小动作链失败时，才退回局部 DOM 定位。

## 区块 -1：目标商品与通道健康

进入条件：

- 用户给出店小秘编辑页链接，或当前 Chrome 中有店小秘 `TikTok > 采集箱` 页面。

动作：

- 使用 Chrome 插件短请求列出标签页，确认店小秘采集箱或编辑页的标题和 URL。
- 如果只有采集箱页面，从当前可见列表选择待编辑商品；明显红线商品可以跳过并记录决定性原因。
- 如果已有编辑页，确认 URL 中的店小秘商品 ID，并避免重复打开同一商品。
- 不用 `body.innerText`、全页 DOM、可见 DOM、整页截图、按钮数量统计或坐标元素探针读取列表。
- 建立队列状态：目标来源为 `dxm_draft_box` 或 `direct_edit_url`；采集表只标记为 `backfill_reference`。

完成校验：

- 已确定一个店小秘商品编辑目标，或已经停止并说明当前可见商品全部不适合编辑。
- Chrome 短请求可用，且当前商品只有一个可操作编辑页。

失败恢复：

- 采集箱列表无法通过插件轻量方式操作时，停止让用户刷新店小秘页面、重启 Chrome 插件会话，或明确授权桌面点击。
- 发现多个同一商品编辑页时，保留当前会话可操作标签；不要继续新开。
- 页面动作第一次超时后只检查标签页标题/URL；第二次同类动作超时后切换低成本路径；第三次前必须停止并让用户选择恢复方式。

## 区块 0：编辑包准备

进入条件：

- 已打开目标店小秘 TikTok 商品编辑页。
- 已确认商品 ID、店铺账号、商品名称或商品键。
- 1688 采购助手已登录，并已找到可信货源；如果未登录或货源不可信，停止。

动作：

- 从店小秘编辑页确认商品名称、店铺名前缀和商品来源；采集表只能辅助匹配，不能替代店小秘当前商品。
- 采集可信 1688 货源字段：链接、供应商、采购价/变种价、起批量、库存、材质、规格/尺寸、重量、发货地、运费/发货时效、评价风险。
- 生成目标分类、必填属性候选、标题、描述、重量尺寸估算和图片处理决策。
- 如果用户要求定价或改价，按 `sku-pricing.md` 为每个已对齐 SKU 生成定价包：采购价、预估海外运费、总费用、建议写入价、预计利润和风险备注。
- 做选品红线复核：大体积、易碎、首饰扣件、侵权、禁售、跨境不适合商品直接停止。

完成校验：

- 编辑包字段齐全，且没有红线风险。
- 标题以店铺前缀开头，例如 `NOMA-`。
- 重量尺寸有来源或估算说明。
- 需要改价时，SKU 定价包已生成，且每个要改价的 SKU 都有明确 1688 对应、采购价、重量尺寸和建议写入价。

失败恢复：

- 缺 1688 登录态：提示用户登录后继续。
- 缺货源关键字段：换 1688 候选；仍缺失则停止，不编造。
- 缺 SKU 定价关键字段：停止定价，不改价格。
- 商品命中红线：停止，不进入页面编辑。
- 采集表匹配不到：可以继续页面编辑，但保存成功后不得回写 `店小秘编辑状态=已编辑`，必须在汇报里说明。

## 区块 1：基本信息

进入条件：

- 编辑包状态为可编辑。
- 当前页是目标商品编辑页。

动作：

- 跳到基本信息区。
- 使用分类搜索选择新版产品分类，必须选择末级类目。
- 分类确认后等待属性刷新。
- 品牌选择 `无品牌`、`No Brand`、`ノーブランド` 或页面同义项。
- 产地、负责方等基础字段如果页面要求，按规则填写安全真实值。

完成校验：

- 页面显示的分类路径与编辑包目标分类一致或足够接近。
- 品牌已显示为无品牌含义。
- 分类后产品属性区和变种属性区已刷新出来。

失败恢复：

- 分类弹窗卡住：关闭弹窗，重新从基本信息区打开分类搜索。
- 分类候选明显不匹配：停止让用户确认，不硬选。
- 属性未刷新：等待后只刷新当前页一次；仍不出现则停止。

## 区块 2：产品属性

进入条件：

- 基本信息区已完成。
- 产品属性区已出现分类后的字段。

动作：

- 只在产品属性区内查找必填字段。
- 按字段名附近控件逐项填写，不扫全页输入框。
- 材质、规格、适用对象、使用场景等优先按 1688 可信货源填写。
- 不确定的非关键可选项可以跳过；不确定的必填项必须使用安全中性值或停止确认。

完成校验：

- 所有可见必填属性都有值。
- 材质、数量、规格没有和 1688 货源、图片、变种冲突。

失败恢复：

- 字段定位超时：重新点产品属性锚点，只读取当前区块可见内容。
- 必填项无法判断：停止，不凭空填写。

## 区块 3：产品信息

进入条件：

- 基本信息和产品属性已完成。
- 标题和描述已在编辑包中生成。

动作：

- 跳到产品信息区。
- 替换标题为编辑包中的日语标题。
- 替换描述为编辑包中的日文描述；优先使用源码模式或编辑器 iframe 的正文区域。
- 不点击页面 AI 生成按钮。

完成校验：

- 标题以店铺前缀开头。
- 标题不含 `送料無料`、`包邮`、官方、大牌、疗效、最强等高风险词。
- 描述不含价格、供应商、销量、站外联系方式或虚假功效。

失败恢复：

- 富文本编辑器无法定位：切换源码模式或重新进入产品信息区。
- 粘贴后内容被页面清洗：读取当前标题/描述，补齐缺失内容。
- 输入动作超时：不要立刻重试同一动作，也不要继续叠加点击、粘贴或按键。先检查当前标签页 URL 和标题；如果插件动作通道仍卡住，刷新当前页或重建浏览器会话，再回到产品信息区续跑。

## 区块 4：图片

进入条件：

- 编辑包已判断是否需要调整主图。

动作：

- 跳到图片信息区。
- 主图清晰、真实、无明显水印且与售卖商品一致时不修改。
- 主图差时，生成不虚构实物的白底或简洁主图，并上传替换或追加到合适位置。

完成校验：

- 主图能清楚表达实际售卖商品。
- 图片不夸大数量、材质、品牌、尺寸或功效。

失败恢复：

- 上传控件不可用：记录图片未处理，继续后续区块或停止等待用户确认。
- 生成图会导致实物不一致：不生成，停止或保留原图。

## 区块 5：变种/SKU

进入条件：

- 分类已刷新出变种属性。
- 编辑包已有 1688 SKU 与店小秘现有变种的对应关系。

动作：

- 只处理变种属性名和值。
- 对齐店小秘已有变种和可信 1688 目标 SKU。
- 不新增无依据 SKU。
- 不修改 SKU 编码、库存、仓库、运费模板或税务字段，除非用户明确要求。
- 价格默认不改；用户要求改价且已确认 SKU 定价报告时，只修改报告中列出的 SKU 价格字段。

完成校验：

- 变种名和值与实际商品一致。
- SKU 数量没有被误删、误增。
- 库存、SKU 编码保持原样。
- 未确认改价时价格保持原样；已确认改价时，价格与 SKU 定价报告一致。

失败恢复：

- SKU 表格过大或定位超时：只用可见行校验数量和关键值，不全表扫描。
- 变种与 1688 货源不一致：停止，不强行映射。

## 区块 6：物流和保修

进入条件：

- 编辑包有重量尺寸来源或估算依据。

动作：

- 跳到物流和保修区。
- 填写重量、长、宽、高。
- 多 SKU 重量不同时，按最重目标 SKU 加包装冗余。
- 保修选择平台/类目允许的安全默认值，不承诺额外保修。

完成校验：

- 重量不低于 1688 包装重量或合理估算值。
- 尺寸包含包装冗余；1688 只给两段规格时，第三段为保守估算并记录说明。

失败恢复：

- 物流字段定位失败：重新点物流锚点，只读取当前区块。
- 无法判断尺寸：停止或使用保守估算并在汇报中标明。

## 区块 7：保存与回写

进入条件：

- 所有必填区块状态为 `done` 或有明确允许的 `skipped`。
- 用户允许保存草稿。

动作：

- 只点击 `保存`。
- 不点击 `保存并移入待发布`、`发布`、`更新`，除非用户明确授权。
- 保存成功后只回写数据库同一条采集记录里的编辑字段：`店小秘编辑状态=已编辑`、`编辑时间`、`编辑标题`、`编辑判断`；不修改采集字段 `商品名称`、`采集状态` 或 `选品判断`。

完成校验：

- 页面提示保存成功，或商品详情页状态能证明草稿已保存。
- 数据库回写命中唯一既有记录。

失败恢复：

- 保存前出现平台确认、资质、付费、发布确认：停止让用户处理。
- 保存失败：记录错误消息，不回写 `店小秘编辑状态=已编辑`。
- 数据库匹配不到或匹配多条：停止，不新增记录。

## 中断续跑

- 续跑时先读取最近区块状态和目标来源，不重新开商品页。
- 已完成区块只做轻量校验，不重复填写。
- 分类被页面重置时，后续属性、变种、物流区块全部退回 `pending`。
- 保存失败但页面字段还在时，从保存区重试；字段丢失时回到对应区块补填。
